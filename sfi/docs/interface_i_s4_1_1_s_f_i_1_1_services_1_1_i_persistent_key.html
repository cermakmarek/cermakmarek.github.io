Detailed Description

Stores a pair of objects to server as a key when information about this instance need to be cached, usually via IS4.SFI.Tools.PersistenceStore<TKey, TValue>. This allows the instance to be reused or discarded as long as the ReferenceKey and DataKey unique identify the entity it describes.

object? IS4.SFI.Services.IPersistentKey.DataKey
get

The equality part of the key, intended to be compared by value (i.e. via System.Object.Equals(object, object)) during caching.

For an entry in an archive, this property could contain the path of the entry within the archive, as long as ReferenceKey identifies the archive itself.

◆ ReferenceKey

object? IS4.SFI.Services.IPersistentKey.ReferenceKey
get

The identity part of the key, intended to be compared by reference (i.e. via System.Object.ReferenceEquals(object, object)) during caching. The logical lifetime of this instance does not exceed the lifetime of this object.

For an entry in an archive, this property could point to the instance describing the archive itself (as long as the same instance is used for all its entries).

Implemented in IS4.SFI.WebApp.BrowserFileInfo, IS4.SFI.Services.RootDirectoryInfo, and IS4.SFI.Services.MemoryStreamFactory.
